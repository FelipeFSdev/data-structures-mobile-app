Singly Linked Lists

Imagine a queue. In this queue, each person has their own characteristics (data) and a coordinate to represent their positions. The coordenate is unknown, so they hold each other's hands, this is a "pointer" to their location.

Knowing the pointer, turns up possible to rearrange the nodes (person) of the queue easily. It's great for situations where you need to add or remove elements often.

In singly lists, "head" refers to the first node in the list and "tail" to the last node.

Each node contain a reference to the next node, except the tail, which points to null (or, the end of the list).

Listas encadeadas simples

Imagine uma fila. Nesta fila, cada pessoa tem suas próprias características (dados) e uma coordenada para representar suas posições. As coordenadas são desconhecidas, então elas seguram as mãos umas das outras. Isto é um "ponteiro" de suas posições.

Conhecendo o ponteiro, é possível reorganizar os nós (pessoas) da fila mais facilmente. Isto é ótimo para situações nas quais você precisa adicionar ou remover elementos frequentemente.

Em listas simples, "cabeça" se refere ao primeiro nó da lista e "cauda" se refere ao último.

Cada nó possui uma referência para o próximo nó, exceto a causa, que aponta para null (ou, o final da lista).
